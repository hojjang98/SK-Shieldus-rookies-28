# Day 33: 개인정보 처리 위탁과 수탁사 관리

## 1. 개인정보 처리 위탁의 개요

### 1.1 위탁이란?

**정의:**
개인정보처리자가 개인정보 처리 업무의 **일부 또는 전부**를 **제3자에게 맡기는** 것

**쉽게:**
- 우리 회사(위탁자)가 할 일을
- 다른 회사(수탁자)에게 시키는 것

**예시:**
```
[위탁자] 쇼핑몰 → [수탁자] 택배회사
개인정보: 이름, 주소, 전화번호 제공
목적: 상품 배송

[위탁자] 은행 → [수탁자] 콜센터
개인정보: 이름, 계좌번호, 전화번호 제공
목적: 고객 상담

[위탁자] 카드사 → [수탁자] 마케팅 대행사
개인정보: 이름, 전화번호, 카드 이용 내역 제공
목적: SMS 마케팅 발송
```

### 1.2 위탁 vs 제3자 제공

**중요한 차이!**

| 구분 | 위탁 | 제3자 제공 |
|------|------|----------|
| 목적 | **위탁자의 업무** 수행 | **제3자 고유 업무** 수행 |
| 지시·통제 | 위탁자가 **통제** | 통제 없음 |
| 책임 | 위탁자가 **책임** | 제3자가 책임 |
| 동의 | **불필요** (공개로 대체) | **필요** |

**예시로 이해:**

**위탁:**
```
쇼핑몰 → 택배회사
- 쇼핑몰이 시킨 일 (배송)
- 쇼핑몰이 책임
- 동의 불필요 (처리방침 공개만)
```

**제3자 제공:**
```
쇼핑몰 → 제휴 카드사
- 카드사 고유 업무 (카드 발급)
- 카드사가 책임
- 동의 필요
```

### 1.3 왜 위탁 관리가 중요한가?

#### (1) 법적 책임

**개인정보보호법 제26조제3항:**
"개인정보처리자는 수탁자가 처리하는 개인정보에 대하여 이 법을 준수할 책임을 진다"

**핵심:**
- 수탁사가 개인정보 유출해도 → **위탁자 책임**
- 수탁사가 법 위반해도 → **위탁자 처벌**

#### (2) 실제 사고 사례

**2014년 카드사 개인정보 유출:**
- 수탁사 직원이 1억건 개인정보 유출
- 카드사(위탁자)가 과징금 수백억원

**2019년 숙박앱 개인정보 유출:**
- 수탁사 서버 해킹
- 위탁자가 책임

#### (3) 실무 현실

**대부분의 기업:**
- 콜센터 → 위탁
- 배송 → 위탁
- IT 운영 → 위탁
- 마케팅 → 위탁

→ **수탁사 없이 사업 불가능**
→ 하지만 **책임은 우리(위탁자)**
→ 따라서 **철저한 관리·감독 필수**

## 2. 법적 근거

### 2.1 개인정보보호법

#### 제26조 (업무위탁에 따른 개인정보의 처리 제한)

**제1항 (위탁 공개):**
"개인정보처리자는 개인정보의 처리 업무를 위탁하는 경우에는 다음 각 호의 내용이 포함된 문서에 의하여야 하며, **개인정보처리방침**을 통하여 공개하여야 한다"
1. 위탁업무 수행 목적 및 범위
2. 수탁자

**제2항 (위탁 계약):**
"개인정보처리자는 위탁하는 업무의 내용과 수탁자를 **개인정보처리방침**을 통하여 **공개**하여야 한다"

**제3항 (위탁자 책임):**
"개인정보처리자는 **수탁자가 처리하는 개인정보에 대하여 이 법을 준수할 책임을 진다**"

**제4항 (재위탁 제한):**
"수탁자가 **재위탁**하는 경우 위탁자의 **동의**를 받아야 한다"

**제5항 (수탁자의 의무):**
"수탁자는 개인정보처리자로부터 위탁받은 업무 범위를 초과하여 개인정보를 이용하거나 제3자에게 제공하여서는 아니 된다"

### 2.2 시행령 제28조 (위탁 계약 시 포함 사항)

**필수 포함 사항:**
1. 위탁업무 수행 목적 외 개인정보 처리 금지
2. 기술적·관리적 보호조치
3. 재위탁 제한
4. 개인정보 관리 현황 점검
5. 수탁자가 준수하여야 할 의무 위반 시 손해배상 등 책임

### 2.3 안전성 확보조치 기준 제11조

**제11조 (개인정보 처리업무 위탁에 따른 관리·감독):**

**제1항:**
"개인정보처리자는 개인정보 처리업무를 위탁하는 경우 다음 각 호의 사항을 준수하여야 한다"

1. 위탁받는 자에 대한 교육
2. **위탁받는 자에 대한 관리·감독**
3. 재위탁 제한

**제2항 (관리·감독 방법):**
"개인정보처리자는 수탁자가 개인정보를 안전하게 처리하는지를 **연 1회 이상 점검**하여야 한다"

**핵심:**
- **연 1회 이상** 점검 **의무**
- 서면, 현장 방문 등 방법 선택 가능

### 2.4 정보통신망법 제25조

**정보통신서비스 제공자의 경우:**
정보통신망법 제25조에도 동일한 위탁 규정

## 3. 위탁 절차

### 3.1 위탁 전 단계

#### (1) 위탁 필요성 검토

**질문:**
- 정말 위탁이 필요한가?
- 자체 수행 불가능한가?
- 개인정보 최소화 가능한가?

#### (2) 수탁자 선정

**평가 항목:**
- [ ] 보안 인증 (ISMS-P, ISO 27001 등)
- [ ] 보안 사고 이력
- [ ] 보안 조직 (CISO, 보안팀)
- [ ] 기술적 보안 (암호화, 접근통제 등)
- [ ] 재무 상태 (손해배상 능력)
- [ ] 레퍼런스

**수탁자 평가표 예시:**
```
평가 항목: 정보보호 인증
□ ISMS-P 보유 (10점)
□ ISO 27001 보유 (5점)
□ 없음 (0점)

평가 항목: 보안 조직
□ CISO 임원급 (10점)
□ CISO 팀장급 (5점)
□ 전담 조직 없음 (0점)

... (생략)

총점: ___점 / 100점
선정 기준: 70점 이상
```

#### (3) 위탁 범위 확정

**명확히 정의:**
- 위탁 업무: "고객 상담"
- 제공 개인정보: "이름, 연락처, 계좌번호"
- 처리 기간: "상담 완료 시까지"
- 보관 기간: "상담 완료 후 즉시 파기"

### 3.2 위탁 계약 체결

#### 필수 포함 조항

**1. 위탁 업무의 목적 및 범위**
```
제○조 (위탁 업무)
수탁자는 위탁자로부터 다음의 업무를 위탁받아 수행한다.
- 위탁 업무: 고객 상담 업무
- 처리 개인정보 항목: 이름, 전화번호, 계좌번호
- 개인정보 주체 수: 약 100만명
```

**2. 목적 외 이용·제공 금지**
```
제○조 (목적 외 이용 금지)
수탁자는 위탁받은 업무 목적 외로 개인정보를 이용하거나 
제3자에게 제공해서는 아니 된다.
```

**3. 재위탁 제한**
```
제○조 (재위탁 제한)
① 수탁자는 위탁자의 사전 서면 동의 없이 제3자에게 
   재위탁할 수 없다.
② 재위탁 시 수탁자는 재수탁자에 대해 본 계약과 동일한 
   의무를 부과하여야 한다.
```

**4. 안전성 확보 조치**
```
제○조 (안전성 확보 조치)
수탁자는 다음의 안전성 확보 조치를 이행하여야 한다.
1. 접근 권한 관리
2. 접근통제 시스템
3. 암호화 (AES-256 이상)
4. 접속기록 보관 (2년)
5. 악성프로그램 방지
6. 물리적 접근통제
7. (이하 생략)
```

**5. 관리·감독**
```
제○조 (관리·감독)
① 위탁자는 수탁자에 대해 연 1회 이상 점검을 실시한다.
② 수탁자는 위탁자의 점검에 협조하여야 한다.
③ 점검 결과 미흡사항 발견 시 수탁자는 즉시 개선하여야 한다.
```

**6. 개인정보 파기**
```
제○조 (개인정보 파기)
① 수탁자는 위탁 업무 종료 시 개인정보를 즉시 파기하여야 한다.
② 파기 완료 후 파기 증명서를 위탁자에게 제출하여야 한다.
```

**7. 손해배상**
```
제○조 (손해배상)
① 수탁자의 귀책사유로 개인정보 유출 시 수탁자는 
   위탁자에게 손해를 배상하여야 한다.
② 배상 한도: 연간 위탁 금액의 ○배
```

**8. 비밀유지**
```
제○조 (비밀유지)
수탁자는 위탁 업무 수행 중 알게 된 개인정보 및 영업비밀을 
누설하거나 업무 외 목적으로 사용해서는 아니 된다.
계약 종료 후에도 동일하게 적용된다.
```

**9. 교육**
```
제○조 (교육)
① 수탁자는 개인정보 처리 직원에 대해 연 1회 이상 
   개인정보 보호 교육을 실시하여야 한다.
② 교육 결과를 위탁자에게 제출하여야 한다.
```

**10. 사고 통지**
```
제○조 (침해사고 통지)
수탁자는 개인정보 침해사고 발생 시 지체 없이 
위탁자에게 통지하여야 한다.
```

#### 계약서 체크리스트

- [ ] 목적 외 이용 금지 조항 있는가?

- [ ] 파기 의무 명시했는가?
- [ ] 손해배상 조항 있는가?

## 4. 수탁사 관리·감독 (핵심!)

### 4.1 법적 의무

**안전성 확보조치 기준 제11조제2항:**
"개인정보처리자는 수탁자가 개인정보를 안전하게 처리하는지를 **연 1회 이상 점검**하여야 한다"

**핵심:**
- **연 1회 이상** 의무
- 방법: 서면, 현장 방문 등 자율

### 4.2 점검 주기

**최소 요구사항:**
- **연 1회 이상**

**실무 권장:**
- 중요 수탁사: **연 2회**
- 일반 수탁사: **연 1회**
- 신규 수탁사: **분기 1회** (첫 해)

### 4.3 점검 방법

#### (1) 서면 점검

**방법:**
- 수탁사에게 **자가진단표** 발송
- 수탁사가 작성 후 제출
- 증빙 자료 첨부

**장점:**
- 비용 낮음
- 시간 절약

**단점:**
- 신뢰도 낮음
- 허위 작성 가능

**적용:**
- 소규모 수탁사
- 위험도 낮은 업무

#### (2) 현장 점검

**방법:**
- 수탁사 사무실 **직접 방문**
- 시스템 확인
- 담당자 인터뷰
- 증빙 자료 확인

**장점:**
- 신뢰도 높음
- 실제 운영 확인 가능

**단점:**
- 비용 높음
- 시간 소요

**적용:**
- 대규모 수탁사
- 중요 개인정보 처리 (민감정보, 고유식별정보 등)
- 고위험 업무

#### (3) 혼합 점검

**방법:**
- 서면 점검 + 선별적 현장 점검

**적용:**
- 실무에서 가장 많이 사용
- 서면으로 1차 점검 후 위험도 높은 항목 현장 확인

### 4.4 점검 항목

#### 점검표 구성

**1. 관리적 보호조치 (20개 항목)**

**(1) 내부 관리계획**
- [ ] 내부 관리계획을 수립했는가?
- [ ] 개인정보 처리방침을 수립했는가?
- [ ] 개인정보 보호책임자를 지정했는가?

**(2) 접근 권한 관리**
- [ ] 개인정보 접근 권한을 최소한으로 부여하는가?
- [ ] 권한을 정기적으로 검토하는가? (분기 1회 이상)
- [ ] 퇴직자 권한을 즉시 삭제하는가?
- [ ] 권한 부여·변경·말소 이력을 기록·보관하는가?

**(3) 교육**
- [ ] 개인정보 보호 교육을 연 1회 이상 실시하는가?
- [ ] 교육 내용이 적절한가?
- [ ] 교육 참석 기록을 보관하는가?

**(4) 위탁 계약 (재위탁)**
- [ ] 재위탁 시 위탁자의 동의를 받았는가?
- [ ] 재수탁자와 별도 계약을 체결했는가?

**2. 기술적 보호조치 (30개 항목)**

- 점검 협조 (자료 제출, 현장 출입)

### 7.3 점검 시

**효율적 점검:**

## 8. 자주 묻는 질문 (FAQ)

### Q1: 수탁사 점검을 안 하면 어떻게 되나요?

**A:**
- 안전성 확보조치 기준 위반
- 과태료 (최대 3천만원)
- 개인정보 유출 시 **위탁자 책임** (관리·감독 소홀)

### Q2: 수탁사가 점검을 거부하면?

**A:**
- 계약 위반
- 계약 해지 사유
- 손해배상 청구 가능

### Q3: 수탁사 점검은 누가 하나요?

**A:**
- **위탁자**가 직접
- 또는 외부 전문기관에 위탁 (보안 컨설팅 회사 등)

### Q4: 점검 결과를 어디 보고하나요?

**A:**
- 법적 보고 의무 없음
- 단, 내부 보고 (경영진)
- 증적 보관 (3년)

### Q5: 수탁사가 재위탁하려면?

**A:**
- **위탁자의 사전 서면 동의** 필수
- 재수탁자도 동일한 점검 필요

### Q6: 수탁사가 여러 개면?

**A:**
- **모든 수탁사** 점검 필요
- 우선순위: 중요도, 위험도 고려

### Q7: 외국 수탁사는?

**A:**
- 동일하게 점검 필요
- 현장 점검 어려우면 서면 + 화상
