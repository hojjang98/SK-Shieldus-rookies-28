# Day 49: 실무 사례 #9 - 옥션·G마켓 개인정보 유출 (2008)

## 1. 사건 개요

### 1.1 기본 정보

회사: 옥션 (Auction), G마켓 (Gmarket)
발생 시기: 2008년 2월
발견: 2008년 2월
유출 규모: 약 1,860만 건 (당시 국내 최대)
유출 방식: 내부자 유출 (중국인 직원)

사건 요약:
2008년 2월, 오픈마켓 업체 옥션과 G마켓의 중국인 협력업체 직원이 고객 개인정보 약 1,860만 건을 불법으로 유출했다. 유출 정보에는 이름, 주민등록번호, 전화번호, 이메일, 주소, 계좌번호 등이 포함되었다. 유출된 정보는 중국으로 반출되어 스팸 발송, 보이스피싱 등에 악용되었다. 개인정보보호법 시행 전이었으나 사회적 충격이 컸으며, 개인정보 보호 제도 강화의 계기가 된 사건이다.

### 1.2 상세 경위

범행 기간: 2007년 말 ~ 2008년 초

범행 주체:
주범: 중국인 A씨 (옥션 협력업체 직원)
공범: 중국인 B씨 (G마켓 협력업체 직원)
중개인: 중국 내 개인정보 거래 브로커

범행 방법:
1. 정당한 업무 권한으로 접근: 고객센터 업무로 고객 정보 조회 권한 보유
2. 대량 조회 및 복사: 수개월간 고객 정보 대량 조회, USB 및 이메일로 복사
3. 중국으로 반출: 유출 정보를 중국으로 전송, 중국 내 개인정보 거래 시장에 판매
4. 스팸 및 보이스피싱 악용: 구매자에게 스팸 이메일, 보이스피싱 전화

유출 데이터:
이름, 주민등록번호, 전화번호, 휴대폰번호, 이메일, 주소, 계좌번호 (일부).

유출 규모:
옥션: 약 1,080만 건
G마켓: 약 780만 건
총: 약 1,860만 건

실제 피해자:
중복 가입 고려 시 약 1,000만명 추정 (대한민국 인구의 20%)

2차 피해:
스팸 이메일 폭주. 보이스피싱 전화 (중국발). "택배 확인하세요", "카드 정지되었습니다" 등. 일부 피해자 금전적 손실.

### 2.2 내부자에 대한 처벌

정보통신망법 적용

법적 근거: 정보통신망법 제49조 (벌칙, 2008년 당시 조문)

조문 (당시):
다음 각 호의 어느 하나에 해당하는 자는 5년 이하의 징역 또는 5천만원 이하의 벌금에 처한다.
업무상 알게 된 개인정보를 누설하거나 권한 없이 다른 사람이 이용하도록 제공한 자.

해당 사항:
중국인 직원 A, B가 업무상 알게 된 고객 정보 누설.

처벌: 5년 이하 징역

형법 적용

업무상 배임죄: 형법 제356조 - 10년 이하 징역

실제 처벌:
중국인 직원 A, B - 출국 후 중국 귀국. 한국 경찰 검거 불가. 중국 공안 협조 거부. 처벌 없음.

국내법 적용 시 예상 처벌: 5년 이하 징역 (정통망법) + 10년 이하 징역 (형법).

### 2.3 옥션·G마켓의 책임

안전성 확보 조치 의무 위반

법적 근거: 정보통신망법 제28조 (개인정보의 보호조치, 2008년 당시)

위반 내용:
1. 접근통제 미흡: 협력업체 직원이 무제한 고객 정보 조회, 업무 목적 외 조회 차단 안 됨, 권한 관리 부실.
2. 로그 관리 미흡: 대량 조회 기록 점검 안 함, 비정상 조회 패턴 탐지 실패, 수개월간 유출 탐지 못함.
3. USB 차단 미흡: 협력업체 직원 PC에서 USB 사용 가능, 개인정보 반출 통제 없음.
4. 이메일 모니터링 부재: 개인정보 포함 이메일 전송 차단 안 됨.
5. 협력업체 관리 부실: 중국인 협력업체 직원 관리 미흡, 보안 교육 부족, 감독 부실.

해당 조항: 정보통신망법 제73조 (벌칙, 당시) - 제28조를 위반한 자는 2년 이하의 징역 또는 2천만원 이하의 벌금에 처한다.

## 3. 기술적 분석

### 3.1 내부자 유출 수법

정당한 권한 악용:
중국인 직원 A, B는 고객센터 업무로 정당한 조회 권한 보유. 고객 문의 응대를 위해 개인정보 접근 필요. 하지만 업무 목적 외 대량 조회.

은밀한 유출:
수개월간 조금씩 조회. 한 번에 대량 조회하면 탐지 위험. 일일 수천 건씩 조회하여 누적.

다양한 반출 경로:
USB 복사. 이메일 전송 (개인 이메일 계정). 클라우드 업로드 (추정).

중국 반출:
유출 정보를 중국으로 전송. 중국 내 개인정보 거래 시장에 판매. 건당 수십 원 ~ 수백 원.

### 3.2 옥션·G마켓의 보안 취약점

(1) 접근통제 미흡

문제:
협력업체 직원이 모든 고객 정보 무제한 조회 가능. 업무 목적 제한 없음. 권한 차등화 없음.

예:
고객센터 직원의 업무: 주문 문의, 배송 문의 응대. 필요 정보: 이름, 주문 내역, 배송 주소. 불필요 정보: 주민번호, 계좌번호. 하지만 모두 조회 가능.

필요한 통제:
최소 권한 원칙. 직무별 차등 권한. 조회 사유 입력 의무화. 민감정보 조회 제한.

(2) 로그 관리 미흡

문제:
수개월간 대량 조회. 명백히 비정상. 하지만 탐지 못함.

원인:
로그 점검 안 함. 비정상 패턴 탐지 시스템 없음. 대량 조회 경고 설정 없음.

필요한 통제:
실시간 모니터링. 대량 조회 자동 경고. 월 1회 로그 점검 (법적 의무). AI 기반 이상 탐지.

(3) USB 차단 미흡

문제:
협력업체 직원 PC에서 USB 사용 가능. 조회 정보를 USB에 복사. 아무런 제약 없음.

필요한 통제:
USB 차단 (물리적 또는 정책). DLP (Data Loss Prevention). 워터마크.

(4) 이메일 모니터링 부재

문제:
개인정보 포함 이메일 전송 차단 안 됨. 직원이 개인 이메일로 정보 전송 가능.

필요한 통제:
이메일 DLP. 개인정보 패턴 감지 및 차단. 외부 이메일 전송 제한.

(5) 협력업체 관리 부실

문제:
중국인 협력업체 직원 관리 미흡. 정규직과 동일 권한. 하지만 보안 교육, 감독 부족. 중국인 직원 = 반출 위험 더 높음 (중국 내 개인정보 시장).

필요한 통제:
협력업체 보안 교육 강화. 법적 구속력 있는 서약서. 권한 최소화. 별도 감독.

### 3.3 사후 옥션·G마켓 보안 강화

2008년 이후 조치:

1. 접근통제 강화
최소 권한 원칙 적용. 직무별 차등 권한. 조회 사유 입력 의무화. 민감정보 조회 제한 (주민번호, 계좌번호).

2. 로그 모니터링 강화
실시간 비정상 조회 탐지. 대량 조회 자동 경고. 주 1회 로그 점검.

3. USB 차단
협력업체 PC USB 완전 차단. DLP 도입. 워터마크 적용.

4. 이메일 모니터링
이메일 DLP. 개인정보 포함 이메일 차단. 외부 전송 승인제.

5. 협력업체 관리 강화
연 4회 보안 교육. 법적 서약서. 권한 최소화. 월 1회 점검.

6. 조직 강화
CISO 신설. 보안팀 증원. 내부 감사 강화.

## 4. 대응 평가

### 4.1 잘한 점

자체 탐지:
외부 신고가 아닌 자체 이상 징후 포착. 내부 조사로 유출 확인.

신속한 신고:
유출 확인 즉시 경찰, 정부 신고.

투명한 공개:
언론 브리핑. 고객 통지. 대표 공식 사과.

### 4.2 미흡한 점

사전 예방 실패:
1. 수개월간 탐지 못함
2. 로그 점검 안 함
3. 접근통제, USB 차단 등 기본 통제 부재

협력업체 관리 부실:
중국인 협력업체 직원 = 고위험. 하지만 관리 미흡.

범인 검거 실패:
중국 귀국 후 검거 불가. 중국 공안 비협조. 처벌 없음.

## 5. 교훈 및 시사점

### 5.1 기술적 교훈

1. 내부자 위협 = 최대 위협

외부 해킹보다 내부자 유출이 더 위험. 정당한 권한 악용. 탐지 어려움.

대응: 최소 권한 원칙. 로그 실시간 모니터링. 제로 트러스트.

2. 협력업체 = 취약점

협력업체 직원 = 보안 취약점. 정규직만큼 관리 안 됨. 특히 외국인 직원 = 반출 위험.

대응: 협력업체 권한 최소화. 별도 관리·감독. 보안 교육 강화.

3. 로그 모니터링 필수

수개월간 대량 조회. 탐지 가능했음. 로그 점검 안 해서 놓침.

대응: 실시간 + 주 1회 점검. 비정상 패턴 자동 탐지.

4. USB 차단

USB = 정보 유출 경로 1순위.

대응: USB 완전 차단. DLP. 워터마크.

5. 이메일 모니터링

개인 이메일로 전송 = 유출 경로.

대응: 이메일 DLP. 외부 전송 승인제.

### 5.2 법률적 교훈

## 9. 실무 적용

### 9.1 외국인 협력업체 직원 관리

추가 고려사항:

신원 조사:
여권 확인. 범죄 경력 조회 (가능 범위). 추천인 확인.

계약 조건:
한국 법 적용 명시. 개인정보 반출 금지 명시. 위반 시 손해배상 (배액).

권한 제한:
외국인 직원은 더 엄격한 권한 제한. 민감정보 접근 불가. 대량 조회 제한 강화.

모니터링 강화:
외국인 직원 로그 더 자주 점검 (주 1회). 이상 징후 즉시 조사.

출국 관리:
계약 종료 시 즉시 권한 회수. 출국 전 데이터 삭제 확인. 서약 준수 서명.

### 9.2 비정상 조회 탐지 기준 (재확인)

경고 기준:

Level 1 (주의):
일 100건 이상 조회. 동일 고객 3회 이상 반복 조회. 업무 시간 외 조회. 보안팀 알림.

Level 2 (경고):
일 500건 이상 조회. 동일 고객 10회 이상 반복 조회. 본인 또는 지인 정보 조회. 팀장 알림 + 본인 경고.

Level 3 (차단):
일 1,000건 이상 조회. 동일 고객 20회 이상 반복 조회. 자동 계정 잠금 + CISO 보고.

협력업체 직원:
정규직보다 더 엄격한 기준 (50% 낮춤). 예: Level 1 = 일 50건, Level 3 = 일 500건.

### 9.3 DLP 정책 (재확인)

차단 대상:

USB:
주민등록번호 패턴. 카드번호 패턴. 대량 개인정보 (10건 이상). 자동 차단 + 보안팀 알림.

이메일:
외부 이메일 (Gmail, Naver 등). 개인정보 포함 이메일. 승인 없이 차단. 승인 프로세스 (상급자).

클라우드:
외부 클라우드 업로드 (Dropbox, Google Drive 등). 개인정보 포함 시 차단.

### 9.4 중국 반출 대응

중국 내 개인정보 시장:
중국은 개인정보 거래 시장 활성화 (암시장). 한국 개인정보 수요 높음 (언어 장벽 낮음).

대응:
중국인 직원 관리 특히 엄격. 중국으로 데이터 전송 차단 (방화벽). 중국 IP 접속 차단 (내부 시스템).

국제 공조:
중국 공안 협조 어려움. 범죄인 인도 협정 부재. 사전 예방이 최선.
