# Day 53: 실무 사례 #13 - 토스 내부자 정보 유출 (2023)

## 1. 사건 개요

### 1.1 기본 정보

회사: 토스 (비바리퍼블리카)
발생 시기: 2023년 3월
발견: 2023년 3월
유출 규모: 약 1,400만 건
유출 방식: 내부자 유출 (전 직원)
유출자: 전 직원 A씨 (데이터 분석가)

사건 요약:
2023년 3월, 토스의 전 직원이 재직 중 고객 개인정보 약 1,400만 건을 불법으로 유출한 사실이 적발되었다. 데이터 분석가였던 A씨는 업무상 접근 권한을 이용하여 고객 정보를 개인 노트북에 복사했으며, 퇴사 후에도 해당 정보를 보유하고 있었다. 유출 정보에는 이름, 주민등록번호 뒷자리, 전화번호 등이 포함되었다. 핀테크 대표 기업의 보안 관리 부실이 드러났으며, 개인정보보호위원회로부터 과징금 60억원이 부과되었다.

### 1.2 상세 경위

범행 기간: 2022년 말 ~ 2023년 초 (재직 중)

범행 주체:
주범: 전 직원 A씨 (30대, 데이터 분석가)
재직 기간: 2020년 ~ 2023년 2월 (약 3년)
퇴사: 2023년 2월

범행 방법:
1. 정당한 업무 권한 보유: 데이터 분석 업무로 고객 DB 접근 권한 보유, 데이터 분석, 통계 작성 등 목적
2. 개인 노트북 반출: 재택근무 시 회사 노트북 + 개인 노트북 동시 사용, 회사 데이터를 개인 노트북에 복사
3. 퇴사 시 미삭제: 2023년 2월 퇴사, 퇴사 시 회사 노트북 반납, 하지만 개인 노트북의 데이터는 삭제 안 함
4. 발각: 2023년 3월 토스 내부 감사에서 발견, A씨 추적 및 확인

유출 데이터:
이름, 주민등록번호 뒷자리 (7자리), 전화번호, 이메일, 토스 가입 일자, 일부 거래 정보 (금액은 제외).

유출 규모:
약 1,400만 건 (토스 전체 가입자 약 2,400만명 중 58%)

용도:
A씨는 유출 정보를 외부 판매하지는 않음 (확인됨). 개인 보유 (이유 불명확 - 이직 후 활용 또는 단순 보유).

### 2.2 내부자 (A씨)에 대한 처벌

개인정보보호법 제70조 (개인정보 누설)

조문: 개인정보처리자인 개인이 업무상 알게 된 개인정보를 누설하거나 권한 없이 다른 사람이 이용하도록 제공한 경우 5년 이하의 징역 또는 5천만원 이하의 벌금에 처한다.

해당 사항:
A씨는 재직 중 업무상 알게 된 고객 정보를 개인 노트북에 복사 (누설). 퇴사 후에도 보유 (권한 없음).

처벌: 5년 이하 징역

신용정보법 제50조 (벌칙)

조문: 업무상 알게 된 개인신용정보를 누설하거나 권한 없이 다른 사람이 이용하도록 제공한 자는 5년 이하의 징역 또는 5천만원 이하의 벌금에 처한다.

해당 사항: 토스 = 금융사 (여신금융). 고객 신용정보 포함 가능.

형법 제356조 (업무상 배임)

조문: 타인의 사무를 처리하는 자가 그 임무에 위배하는 행위로써 재산상의 이익을 취득하거나 제3자로 하여금 이를 취득하게 하여 본인에게 손해를 가한 때에는 10년 이하의 징역 또는 3천만원 이하의 벌금에 처한다.

해당 사항: 토스 직원으로서 회사 데이터 무단 반출 = 임무 위배.

실제 처벌 (1심):

징역 1년 6개월 (집행유예 2년)
벌금 1천만원
사회봉사 80시간

이유 (판결문):
- 외부 판매 안 함 (정상 참작)
- 초범
- 반성
- 하지만 1,400만 건 = 막대한 규모
- 토스 = 금융사, 신뢰 중요

### 2.3 토스의 책임

안전성 확보 조치 의무 위반

법적 근거: 개인정보보호법 제29조 (안전성 확보 조치)

위반 내용:
1. 재택근무 통제 미흡: 개인 노트북 사용 허용, 회사 데이터 개인 기기 복사 차단 안 됨, DLP (Data Loss Prevention) 미흡
2. 접근통제 미흡: 데이터 분석가에게 전체 고객 DB 접근 허용, 업무 목적 외 접근 차단 안 됨, 최소 권한 원칙 미적용
3. 로그 관리 미흡: 대량 다운로드 기록 점검 안 함 (재직 중), 퇴사 후 정기 감사에서 발견 (지연)

해당 조항: 개인정보보호법 제75조 (과징금) - 안전성 확보 조치를 하지 아니하여 개인정보를 유출당한 자에게는 매출액의 3% 이하 과징금 부과 가능.

토스 = 여신금융회사. 신용정보 보호 의무. 동일한 위반.

## 3. 기술적 분석

### 3.1 재택근무 보안 취약점

문제 상황:

재택근무 시:
직원이 집에서 근무. 회사 노트북 + 개인 노트북 동시 사용. 회사 데이터를 개인 노트북에 복사 가능. 회사에서 통제 어려움.

A씨 사례:
업무상 고객 데이터 조회. 분석 결과를 개인 노트북에 저장 (편의). 반복적 복사. 회사는 탐지 못함.

필요한 통제:

1. 개인 기기 사용 금지
회사 노트북만 사용. 개인 노트북 업무 사용 금지. BYOD (Bring Your Own Device) 금지.

2. DLP (Data Loss Prevention)
개인정보 포함 파일 개인 기기 복사 차단. USB, 이메일, 클라우드 등 모든 경로 차단. 패턴 감지 (주민번호, 전화번호 등).

3. VDI (Virtual Desktop Infrastructure)
가상 데스크톱. 서버에서 화면만 전송. 로컬 저장 불가. 복사 붙여넣기 차단.

4. 접속 로그 모니터링
재택근무 중 접속 기록. 대량 다운로드 탐지. 비정상 패턴 경고.

### 3.2 토스의 보안 취약점

(1) 재택근무 통제 미흡

문제:
코로나19 이후 재택근무 확대 (2020~). 개인 노트북 사용 허용 (편의성). 회사 데이터 개인 기기 복사 차단 안 됨.

원인:
DLP 미구축 또는 미흡. 재택근무 보안 정책 부족. 개인 기기 관리 안 함.

필요한 대책:
DLP 전면 도입. VDI 도입. 개인 기기 사용 금지. 재택근무 보안 정책 수립.

(2) 접근통제 미흡

문제:
데이터 분석가에게 전체 고객 DB 접근 허용. 1,400만 건 전체 조회 가능. 업무상 필요한 건수보다 과도.

필요한 대책:
최소 권한 원칙. 업무별 필요한 데이터만 접근. 샘플링 데이터 제공 (전체 불필요). 조회 건수 제한.

(3) 로그 관리 미흡

문제:
재직 중 대량 다운로드 탐지 못함. 퇴사 후 정기 감사에서 발견 (수개월 지연).

필요한 대책:
실시간 로그 모니터링. 대량 다운로드 자동 경고. 주 1회 로그 점검. AI 기반 이상 탐지.

(4) 퇴사자 관리 부실

문제:
퇴사 시 회사 노트북만 반납. 개인 노트북 점검 안 함. 데이터 삭제 확인 안 함.

필요한 대책:
퇴사 시 개인 기기 점검. 회사 데이터 삭제 확인. 서약서 징구 (데이터 삭제 확인). 퇴사 후 1개월 내 추적 조사.

### 3.3 사후 토스 보안 강화

2023년 이후 조치:

1. DLP 전면 도입
모든 PC에 DLP 설치. 개인정보 패턴 감지. USB, 이메일, 클라우드 차단. 재택근무 PC도 포함.

2. VDI 도입
데이터 분석가는 VDI만 사용. 로컬 저장 불가. 화면만 전송.

3. 접근통제 강화
최소 권한 원칙. 데이터 분석가는 샘플링 데이터만. 전체 DB 접근 시 승인 필요. 조회 건수 제한 (일 1,000건 등).

4. 로그 모니터링 강화
실시간 모니터링. 대량 다운로드 자동 경고. 주 1회 → 일 1회 점검. AI 이상 탐지.

5. 퇴사자 관리 강화
퇴사 시 개인 기기 점검 (의무). 회사 데이터 삭제 확인. 서약서 징구. 퇴사 후 1개월 내 추적 조사.

6. 재택근무 정책 개편
개인 노트북 업무 사용 금지. 회사 노트북만 사용. VPN + DLP 필수.

7. 조직 강화
CISO 권한 강화. 보안팀 인력 증원. 외부 보안 전문가 자문.

3. DLP 미구축
4. 퇴사자 관리 부실

늦은 발견:
재직 중 탐지 못함. 퇴사 후 발견 (수개월 지연).

핀테크 = 높은 기준:
토스 = 핀테크 대표 기업. 금융사 수준 보안 필요했으나 미흡.

## 5. 교훈 및 시사점

### 5.1 기술적 교훈

1. 재택근무 = 보안 취약점

개인 기기 사용 = 통제 어려움. 데이터 반출 쉬움.

대응: DLP 필수. VDI. 개인 기기 사용 금지. 재택근무 보안 정책.

2. 최소 권한 원칙

전체 DB 접근 = 위험. 업무 목적만 접근.

대응: 샘플링 데이터. 조회 건수 제한. 승인제.

3. 퇴사자 = 고위험

퇴사 직전 데이터 반출 위험 높음. 퇴사 후 악용 가능.

대응: 퇴사 시 개인 기기 점검. 데이터 삭제 확인. 추적 조사.

4. 로그 모니터링 실시간

재직 중 탐지 못함 = 실패. 실시간 필요.

대응: 일 1회 점검. 대량 다운로드 자동 경고. AI 이상 탐지.

5. 핀테크 = 금융권 수준

핀테크 = 금융 + IT. 일반 IT 기업보다 높은 보안 필요.

대응: 금융권 보안 가이드 준수. 정기 감사.

### 5.2 법률적 교훈

1. 내부자도 5년 이하 징역

개인정보 누설 = 5년 이하 징역. 초범, 외부 판매 안 해도 처벌.

A씨: 징역 1년 6개월 (집행유예). 벌금 1천만원.

교훈: 내부자도 엄중 처벌. 데이터 반출 절대 금지.

2. 회사도 과징금

토스 = 안전조치 의무 위반. 과징금 60억원.

교훈: 회사도 책임. 직원 관리 철저히.

3. 재택근무 = 회사 책임

재택근무 허용한 회사 = 통제 책임. DLP 등 필수.

교훈: 재택근무 보안 투자 필수.

4. 핀테크 규제 강화

토스 사건 이후 핀테크 규제 강화 논의. 금융권과 동일한 보안 기준 요구.

교훈: 핀테크도 금융사. 높은 기준.

### 5.3 경영적 교훈

1. 과징금 + 소송 = 막대한 비용

직접 비용 200억원 이상 (소송 제외). 소송 결과 따라 수백억원 추가.

사전 투자 (추정): DLP 5억원, VDI 10억원, 관리 강화 5억원. 총 20억원.

200억원 vs 20억원 = 10배 차이.

2. 평판 손실

토스 = 핀테크 대표. 보안 사고 = 신뢰 추락. 경쟁사 (카카오뱅크 등)로 이탈.

교훈: 평판 회복 어려움. 사전 예방.

3. 핀테크 = 높은 기준

고객 신뢰 = 핀테크 생명. 보안 사고 = 치명적.

교훈: 보안 투자 아끼면 안 됨.

4. 재택근무 시대

코로나19 이후 재택근무 일반화. 보안 투자 필수.

교훈: 재택근무 = 새로운 위협. 대응 필요.

### 5.4 사회적 영향

핀테크 보안 점검:
토스 사건 이후 전 핀테크 업계 보안 점검. 카카오뱅크, 케이뱅크 등 DLP 도입 가속화.

재택근무 보안 인식:
재택근무 = 보안 취약점 인식. 기업들 DLP, VDI 도입 증가.

퇴사자 관리 강화:
전 업계 퇴사자 관리 절차 강화. 개인 기기 점검 일반화.

## 9. 실무 적용

### 9.1 DLP 구축 가이드

DLP 기능:

1. 개인정보 패턴 감지
주민등록번호: 123456-1234567
전화번호: 010-1234-5678
카드번호: 1234-5678-9012-3456
이메일: xxx@xxx.com

2. 차단 대상
USB 복사
이메일 첨부
클라우드 업로드 (Dropbox, Google Drive 등)
화면 캡처
출력

3. 정책 설정
개인정보 10건 이상 포함 시 차단
또는 1건이라도 차단 (엄격)
승인 워크플로우 (상급자 승인)

4. 모니터링
차단 이벤트 로그
실시간 알림 (보안팀)

주요 솔루션:
Symantec DLP, McAfee DLP, Forcepoint DLP, 파수 (Fasoo) 등.

### 9.2 VDI 구축 가이드

VDI 구조:

서버: 중앙 서버에 가상 데스크톱
클라이언트: 화면만 전송 (원격 제어)
데이터: 서버에만 저장, 클라이언트는 저장 불가

장점:
데이터 반출 불가
보안 중앙 관리
재택근무 적합

단점:
네트워크 의존 (느릴 수 있음)
초기 구축 비용 (서버, 라이선스)

주요 솔루션:
VMware Horizon, Citrix Virtual Apps and Desktops, Microsoft Azure Virtual Desktop 등.

### 9.3 퇴사자 관리 절차

퇴사 1주일 전:
권한 제한 (민감 정보 접근 차단)
데이터 접근 로그 집중 모니터링

퇴사 당일:
즉시 모든 권한 회수 (계정 비활성화)
회사 노트북 반납
개인 노트북 점검 (IT팀)
회사 데이터 삭제 확인
서약서 징구 (데이터 삭제, 비밀 유지)

퇴사 후 1개월:
접근 로그 재검토
비정상 접근 확인
필요 시 추적 조사 (개인 기기 재점검 요청)

### 9.4 재택근무 보안 정책 예시

정책 내용:

1. 개인 노트북 업무 사용 금지
회사 노트북만 사용
개인 기기에 회사 데이터 저장 금지

2. VPN 필수
재택근무 시 VPN 접속 필수
VPN 없이 회사 시스템 접근 불가

3. 공용 WiFi 금지
집 WiFi만 사용
카페, 공공장소 WiFi 금지

4. 화면 보안
화면 보호기 (5분)
타인이 볼 수 없는 공간에서만 작업

5. 위반 시 징계
경고 → 정직 → 해고
심각한 경우 형사 고소
