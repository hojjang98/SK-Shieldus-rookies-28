# Day 46: 실무 사례 #6 - 카드3사 개인정보 유출 사건 (2014)

## 1. 사건 개요

### 1.1 기본 정보

회사: KB국민카드, 롯데카드, NH농협카드
발생 시기: 2013년 12월 ~ 2014년 1월
발견: 2014년 1월 8일
유출 규모: 약 1억 400만 건 (역대 최대 건수)

사건 요약:
신용정보회사 직원과 카드사 협력업체 직원이 공모하여 3개 카드사의 고객 개인정보 1억 400만 건을 불법으로 유출했다. 유출 정보에는 이름, 주민등록번호, 카드번호, 유효기간, 전화번호 등이 포함되었으며, 대출 모집인 등에게 판매되었다. 국내 역사상 건수 기준 최대 규모의 개인정보 유출 사건이다.

### 1.2 상세 경위

범행 기간: 2013년 12월 ~ 2014년 1월 (약 1개월)

범행 주체:
1. 주범 A: 코리아크레딧뷰로 (KCB, 신용정보회사) 직원
2. 공범 B: KB국민카드 협력업체 직원
3. 공범 C: 롯데카드 협력업체 직원
4. 공범 D: NH농협카드 협력업체 직원
5. 중개인 E: 대출 모집인
총 5명 및 추가 공모자

범행 방법:
1. 협력업체 직원 B, C, D가 각 카드사 시스템에 정당한 업무 권한으로 접근
2. 고객 정보를 조회하여 USB에 복사
3. 주범 A (KCB 직원)에게 전달
4. 주범 A가 KCB 시스템에서 추가 정보 (신용등급, 연체 정보 등) 결합
5. 중개인 E를 통해 대출 모집인, 보험 설계사 등에게 판매
6. 건당 200원~300원에 판매, 총 수익 약 2억원

유출 데이터:
이름, 주민등록번호, 전화번호, 주소, 카드번호, 유효기간, CVC (카드 뒷면 3자리), 신용등급, 연체 정보, 대출 정보.

유출 규모:
KB국민카드: 약 5,400만 건
롯데카드: 약 3,200만 건
NH농협카드: 약 1,800만 건
총: 약 1억 400만 건

실제 피해자:
중복 가입 고려 시 약 2,000만명 추정 (대한민국 인구의 40%)

### 1.3 타임라인

(2) 카드사 및 KCB의 책임

위반 내용:
1. 접근통제 미흡
협력업체 직원이 무제한 고객 정보 조회 가능. 업무 목적 외 조회 차단 장치 없음. 권한 관리 부실.

2. 로그 관리 미흡
개인정보 접근 기록 점검 안 함. 비정상 조회 (대량, 반복) 탐지 실패. 월 1회 점검 의무 위반.

3. USB 차단 미흡
협력업체 직원 PC에서 USB 사용 가능. 개인정보 반출 통제 없음.

4. 협력업체 관리 부실
협력업체 직원에 대한 보안 교육 미흡. 보안 서약서 형식적. 감독 부실.

해당 조항:
신용정보법 제52조 (과태료): 제32조를 위반한 자에게는 5천만원 이하의 과태료를 부과한다.
개인정보보호법 제71조 (벌칙): 제29조를 위반하여 안전성 확보에 필요한 조치를 하지 아니하여 개인정보를 유출당한 자는 2년 이하의 징역 또는 2천만원 이하의 벌금에 처한다.

## 3. 기술적 분석

### 3.1 내부자 유출의 전형

정당한 권한 악용:
협력업체 직원 B, C, D는 업무상 고객 정보 조회 권한 보유. 콜센터, 마케팅 업무로 정당하게 접근. 하지만 업무 목적 외 대량 조회 및 복사.

탐지 어려움:
정상 업무와 구분 어려움. 시스템은 정당한 접근으로 인식. 로그 분석 안 하면 발견 불가.

장기간 지속:
약 1개월간 지속적 유출. 1억 건 이상 대량 유출. 자동 탐지 시스템 없었음.

### 3.2 카드사의 보안 취약점

(1) 접근통제 미흡

문제:
협력업체 직원이 업무 목적 없이도 모든 고객 정보 무제한 조회 가능.

예:
콜센터 직원의 업무: 고객 문의 응대. 필요 정보: 이름, 전화번호, 최근 거래 내역. 불필요 정보: 주민번호, 카드번호, CVC. 하지만 모두 조회 가능했음.

필요한 통제:
1. 최소 권한 원칙: 직원별로 필요한 정보만 조회. 콜센터 직원은 주민번호, 카드번호 조회 불가.
2. 목적 기반 접근통제: 조회 시 사유 입력. 부당한 조회 차단.
3. 조회 건수 제한: 일일 조회 건수 상한. 초과 시 자동 차단.

(2) 로그 관리 미흡

문제:
1개월간 1억 건 조회. 평균 일 300만 건 이상. 명백히 비정상. 하지만 탐지 못함.

원인:
로그 점검 안 함. 법적 의무 (월 1회) 위반. 비정상 패턴 탐지 시스템 없음.

필요한 통제:
1. 실시간 모니터링: 비정상 조회 패턴 자동 탐지. 대량 조회, 반복 조회 알림.
2. 월 1회 로그 점검 (법적 의무): 개인정보 접근 기록 검토. 이상 징후 조사.
3. AI 기반 이상 탐지: 정상 패턴 학습. 비정상 패턴 자동 감지.

(3) USB 차단 미흡

문제:
협력업체 직원 PC에서 USB 사용 가능. 조회한 정보를 USB에 복사. 아무런 제약 없음.

필요한 통제:
1. USB 차단: 협력업체 PC USB 포트 물리적 차단. 또는 정책으로 차단.
2. DLP (Data Loss Prevention): 개인정보 포함 파일 USB 복사 차단. 개인정보 패턴 감지 및 차단.
3. 워터마크: 화면에 조회자 정보 표시. 누가 조회했는지 추적 가능.

(4) 협력업체 관리 미흡

문제:
협력업체 직원에 대한 보안 관리 부실. 정규직과 동일한 권한. 하지만 보안 교육, 감독 미흡.

필요한 통제:
1. 보안 교육: 연 4회 이상. 개인정보 보호, 처벌 사례 공유.
2. 보안 서약서: 법적 구속력 있는 서약서. 위반 시 손해배상 조항.
3. 감독 강화: 협력업체 직원 조회 기록 별도 점검. 정규직보다 더 엄격.
4. 권한 최소화: 협력업체 직원은 최소 권한만. 민감정보 조회 제한.

### 3.3 사후 카드사 보안 강화

2014년 이후 조치:

1. 접근통제 강화
최소 권한 원칙 적용. 직무별 차등 권한. 조회 사유 입력 의무화. 조회 건수 제한 (일 100건 등).

2. 로그 모니터링 강화
실시간 비정상 조회 탐지. AI 기반 이상 패턴 분석. 월 1회 → 주 1회 점검.

3. USB 차단
협력업체 PC USB 완전 차단. DLP 도입. 워터마크 적용.

4. 협력업체 관리 강화
연 4회 보안 교육. 법적 구속력 있는 서약서. 점검 강화 (월 1회). 권한 최소화.

5. 조직 강화
CISO 권한 강화. 보안팀 인력 증원. 내부 감사 강화.

## 4. 대응 평가

### 4.1 잘한 점

신속한 신고:
유출 확인 즉시 금융감독원 신고. 법적 의무 준수.

투명한 공개:
언론 브리핑. 홈페이지 공지. 고객 통지 (SMS, 우편). 대표 공식 사과.

범인 검거 협력:
경찰 수사 적극 협조. 범인 전원 검거.

### 4.2 미흡한 점

사전 예방 실패:
1. 1개월간 탐지 못함. 1억 건 유출 후 발견.
2. 로그 점검 안 함. 법적 의무 위반.
3. 접근통제, USB 차단 등 기본 통제 부재.

협력업체 관리 부실:
협력업체 직원 = 취약점. 정규직과 동일 권한. 하지만 관리·감독 미흡.

초기 대응:
금융감독원이 먼저 이상 징후 포착. 카드사 자체 탐지 실패.

## 5. 교훈 및 시사점

### 5.1 기술적 교훈

1. 내부자 위협은 가장 위험

외부 해킹보다 내부자 유출이 더 위험. 정당한 권한 악용. 탐지 어려움.

대응: 최소 권한 원칙. 로그 실시간 모니터링. 제로 트러스트 (Zero Trust).

2. 협력업체는 취약점

협력업체 직원 = 보안 취약점. 정규직만큼 관리 안 됨. 하지만 동일한 권한.

대응: 협력업체 직원 권한 최소화. 별도 관리·감독. 보안 교육 강화.

3. 로그 모니터링 필수

개인정보 접근 기록 점검 의무 (월 1회). 실무에서는 실시간 + 주 1회 권장.

비정상 패턴: 대량 조회, 반복 조회, 업무 시간 외 조회. 자동 탐지 및 경고.

4. USB 차단

USB = 정보 유출 경로 1순위. 개인정보 취급 PC는 USB 완전 차단.

대안: DLP, 워터마크, 화면 캡처 방지.

5. 최소 권한 원칙

직원은 업무에 필요한 최소한의 정보만 조회. 콜센터 직원이 주민번호 조회 필요한가? 카드번호 전체 필요한가?

권한 분리: 조회, 수정, 삭제 권한 분리. 승인 단계 추가.

### 5.2 법률적 교훈

1. 내부자도 엄중 처벌

개인정보 누설: 5년 이하 징역. 업무상 배임: 10년 이하 징역.

실제 처벌: 징역 3년 6개월 실형. 내부자라도 엄중 처벌.

2. 회사도 책임

안전조치 의무 위반. 내부 통제 미흡 = 회사 책임.

과징금: 57억원. 형사 처벌 가능.

3. 협력업체 직원도 동일 책임

협력업체 직원도 개인정보 취급자. 동일한 법적 의무. 동일한 처벌.

회사는 협력업체 관리 책임. 관리 부실 시 회사도 처벌.

4. 법 개정의 계기

카드3사 사건 이후 법 개정. 과징금 상향 (매출액 3%), 내부 통제 의무 강화, 협력업체 관리 의무 법제화.

### 5.3 경영적 교훈

1. 내부 통제 투자 필수

카드3사 총 손실: 약 600억원 이상 (직접) + 수천억원 (간접).

사전 투자 (추정): 접근통제 강화 10억원, DLP 도입 5억원, 협력업체 관리 5억원. 총 20억원.

600억원 vs 20억원 = 30배 차이.

2. 협력업체 관리는 필수

협력업체 = 보안 취약점. 하지만 현실적으로 협력업체 없이 불가능.

대응: 협력업체도 정규직과 동일한 보안 기준. 관리·감독 강화. 계약서에 보안 의무 명시.

3. 평판 회복 어려움

카드3사는 사건 후 고객 이탈. 신뢰 추락. 경쟁사로 고객 이동.

교훈: 한 번의 사고가 수십 년 쌓은 신뢰 무너뜨림.

### 5.4 사회적 영향

개인정보 유통 시장 단속 강화:
카드3사 사건 이후 개인정보 불법 거래 단속 강화. 대출 모집인, 보험 설계사 등 단속. 개인정보 매매 근절 노력.

주민등록번호 수집 제한:
주민번호 유출 위험 인식. 수집 자제 움직임. 대체 인증 수단 확산 (아이핀, 휴대폰).

금융권 내부 통제 강화:

## 9. 실무 적용

### 9.1 협력업체 관리 가이드

계약 단계:
보안 의무 조항 명시. 개인정보 보호 교육 의무. 위반 시 손해배상 (계약 해지 + 배상).

운영 단계:
연 4회 보안 교육. 법적 구속력 있는 서약서 징구. 월 1회 로그 점검 (별도). 분기 1회 현장 점검.

권한 관리:
최소 권한만 부여. 민감정보 (주민번호, 카드번호 등) 조회 제한. 승인 없이 조회 불가.

모니터링:
협력업체 직원 조회 건수 일일 모니터링. 비정상 패턴 즉시 조사. 정규직보다 더 엄격.

### 9.2 비정상 조회 탐지 기준

경고 기준:

Level 1 (주의):
일 100건 이상 조회. 동일 고객 3회 이상 반복 조회. 업무 시간 외 조회 (야간, 주말). 보안팀 알림.

Level 2 (경고):
일 500건 이상 조회. 동일 고객 10회 이상 반복 조회. 본인 또는 가족 정보 조회. 팀장 알림 + 본인 경고.

Level 3 (차단):
일 1,000건 이상 조회. 동일 고객 20회 이상 반복 조회. 자동 계정 잠금 + CISO 보고.

### 9.3 DLP 정책 예시

차단 대상 (USB, 이메일, 메신저):

주민등록번호 패턴: 123456-1234567 (6자리-7자리).
카드번호 패턴: 1234-5678-9012-3456 (16자리, 하이픈 포함).
CVC 패턴: 카드번호 + 3자리 숫자.
대량 개인정보: 이름 + 전화번호 10건 이상.

자동 차단 + 보안팀 알림 + 로그 기록.
