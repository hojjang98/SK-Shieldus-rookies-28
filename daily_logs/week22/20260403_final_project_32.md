## 2. 실습 내용 정리

### 실습: PPT 목차 구성 설계 (팀 회의 기반)

**목표:** 최종 발표용 PPT의 논리적 목차 체계 확정

**실습 환경:**
- 팀 회의 (2차 PPT 회의, 전체 구성원 참여)
- 기존 PPT 초안 검토
- 멘토 제공 컨설팅 방법론 이미지 참고

**확정된 목차 구조:**
```
1. 프로젝트 개요
   1.1 개인정보 위수탁 관리체계 소개
   1.2 프로젝트 배경 및 선정 이유
   1.3 항공사 선정 이유
   1.4 프로젝트 주제 및 목표
   1.5 현업 위수탁 관리 체계 프로세스
   1.6 프로젝트 진행 일정

2. 프로젝트 소개 및 구조
   2.1 위탁사 소개
   2.2 수탁사 선정 이유
   2.3 수탁사 목록 및 소개
   2.4 위탁사·수탁사 관계도
   2.5 흐름도 (컨설팅 방법론 기반)

3. 프로젝트 내용
   3.1 사업 제안서
   3.2 수탁사 현황 분석
   3.3 수탁사별 점검 계획서
   3.4 수탁사별 체크리스트
   3.5 서면 점검
   3.6 현장 점검
   3.7 수탁사별 증적 자료

4. 프로젝트 결과
   4.1 점검 결과 기준 설명 (취약/부분양호/양호)
   4.2 등급 점수 기준 + 점검결과보고서
   4.3 개선방안 보고서
   4.4 통합 점검 결과 보고서

5. 사용 툴 및 산출물
6. 프로젝트 결론
7. 프로젝트 회고
8. 자료 출처
9. Q&A
```

**분석 포인트:**
```
확인 항목:
✓ 프로젝트 주제명이 명확히 드러나는 페이지 별도 필요
✓ 현업 프로세스와 팀 프로세스의 유사성을 구조적으로 드러낼 것
✓ 04. 점검 결과 분석 챕터는 삭제 — 04. 프로젝트 결과로 통합
✓ WBS 페이지 삭제 (불필요)
✓ AI 프롬프트 섹션은 5장으로 이동 (발표 흐름 정리)
```

**보안 인사이트:**
- 발표 구성도 컨설팅 산출물과 동일한 논리 흐름을 따라야 설득력이 생김
- '현업과 동일한 프로세스'를 차별점으로 내세울 때, 실제 현업 자료 인용이 필수 (SK쉴더스 자료 활용 검토)
- 수치/통계 미비 시 주장의 신뢰도 급락 — 배경 및 선정 이유 슬라이드에 구체적 수치 조사 필요

## 6. 배운 점 및 인사이트

### 새로 알게 된 점

- **목차 = 논리 구조:** PPT 목차를 짜는 것 자체가 컨설팅 프로세스를 재현하는 작업임을 다시 확인함
- **CPO와 CISO 구분:** 같은 보안 관련 최고책임자라도 법적 근거와 역할이 다르며, 체크리스트 내 혼용은 문서 신뢰도를 떨어뜨림
- **증적 자료의 법적 의미:** 점검 결과는 증적 없이는 주장에 불과하며, 실무에서는 증적 보관이 감사 대응의 핵심 수단임
- **발표 구성의 차별점:** '현업과 동일한 프로세스를 따랐다'는 사실을 구조적으로 드러내는 것이 이 프로젝트의 가장 강력한 차별점
- **수치 없는 주장의 한계:** 중간 발표 피드백에서 지적된 것처럼, 배경 및 선정 이유는 반드시 구체적인 수치·출처·사례로 뒷받침되어야 함

### 이전 학습과의 연결고리

- **개인정보보호법 학습과 연계:** CPO/CISO 구분은 법령 체계에서 직접 도출되는 개념 — 단순 용어 문제가 아닌 법적 책임 주체 문제
- **체크리스트 설계 원칙 확장:** 수탁사별 등급 기준 통일에서, 이번에는 직책명 표기 통일로 동일한 문서 일관성 원칙이 재적용됨
- **문서 품질 관리 → 발표 품질 관리:** 보고서에서 적용하던 '일관성·명확성·근거' 원칙이 PPT 슬라이드 구성에도 동일하게 적용됨

### 실무 적용 아이디어

**보안 전문가 관점:**
- **컨설팅 방법론 시각화:** 현업 컨설팅 절차를 플로우 차트로 표현하고 팀 프로젝트와 병렬 비교하면 청중 이해도 극대화
- **등급 기준 선제 설명:** 결과를 보여주기 전에 판정 기준부터 설명하는 순서가 발표 신뢰도에 직접 영향
- **3자 기대효과 분리 서술:** 컨설팅 결과의 의미를 이해관계자별로 분리 서술하는 습관은 실제 보고서 작성에도 그대로 적용 가능

**프로젝트 관리 관점:**
- **역할 분담 명확화:** PPT 팀과 보고서 리뷰 팀을 병렬 운영하는 구조 자체가 실무 프로젝트 분업 방식과 유사
- **피어리뷰 기반 수정:** 어제 피어리뷰 결과를 오전 중 반영 완료하는 일정 관리는 마감 역산 계획의 좋은 실습

## 7. Quick Reference

### 최종 PPT 챕터 요약

```
1장 프로젝트 개요     → 배경/선정이유/목표/프로세스/일정
2장 소개 및 구조      → 위탁사·수탁사 관계, 흐름도
3장 프로젝트 내용     → 사전준비~증적자료 (수행 전 과정)
4장 프로젝트 결과     → 결과 기준 설명 + 보고서 + 개선방안 + 통결보
5장 사용 툴 및 산출물 → 도구소개 + AI 프롬프트 + 산출물 목록
6장 결론              → 기대효과 + 인사이트
7장 회고              → 역할분담 + 소감 + 한계점
8장 자료 출처
9장 Q&A
```

### 핵심 체크리스트 이슈 요약표

| 구분 | 항목 | 핵심 키워드 | 주요 내용 | 적용 방법 |
|:---:|:---|:---|:---|:---|
| 문서 일관성 | CPO/CISO 표기 | 직책명 통일 | 해랑금융 체크리스트 수정 완료 | CISO로 통일, 시나리오에 두 직책 지정 명시 |
| 점검 등급 | 취약/부분양호/양호 | 0 / 0.7 / 1 | 기준 먼저, 결과 후 설명 순서 유지 | 4.1 슬라이드에서 선제적으로 기준 정의 |
| 배경 근거 | 수치 및 출처 | 침해 통계, 기사 | 위수탁 관련 실제 사고 사례 필요 | 담당자 지정하여 조사 후 1.2 슬라이드에 반영 |
