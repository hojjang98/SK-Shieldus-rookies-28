## 2. 실습 내용 정리

### 활동 1: 중간 발표 PPT 구성 설계

**목표:** 3월 26일(목) 중간 발표를 위한 PPT 스토리라인 확정

**구성 원칙:**
```
확인 항목:
✓ 프로젝트의 Why → What → How 순서로 논리 흐름 구성
✓ 흐름도는 점검 방법론 5단계를 한눈에 볼 수 있도록 시각화
✓ 현재 진행도는 WBS 기준으로 완료/진행 중/예정 구분
✓ 예시 산출물은 완성도가 높은 것으로 선별
✓ 형상관리 슬라이드는 팀 운영의 체계성을 어필하는 방향으로
✓ Q&A 대비: 점검 방법론, 관련 법령, 위험관리 답변 준비
```

**보안 인사이트:**
- 중간 발표는 내용 완성도보다 프로세스의 논리적 흐름을 보여주는 자리
- 형상관리 슬라이드는 팀 관리 역량을 증명하는 실질적 근거
- 예시 산출물 선택 시 체크리스트·진단결과보고서처럼 점검 방법론이 직접 드러나는 것이 효과적

## 6. 배운 점 및 인사이트

### 새로 알게 된 점

- **중간 발표의 목적:** 완성된 결과를 보여주는 것이 아니라, 프로젝트가 올바른 방향으로 올바른 절차를 밟고 있음을 증명하는 자리. 프로세스 중심 관점(Day 96)이 발표 구성에도 그대로 적용됨
- **형상관리의 발표 가치:** WBS·회의록·노션·드라이브 같은 관리 도구를 단순 나열이 아닌, 프로젝트를 어떻게 통제하고 있는지를 보여주는 증거로 제시하는 것이 핵심

### 이전 학습과의 연결고리

- **점검 방법론 5단계(Day 93)와 연계:** 흐름도 슬라이드의 뼈대가 그대로 활용됨
- **프로세스 페르소나(Day 96)와 연계:** 발표 전체 스토리라인이 "우리는 프로세스를 따랐다"는 메시지를 중심으로 구성됨

### 실무 적용 아이디어

**보안 전문가 관점:**
- **중간 보고의 실무 의미:** 실제 컨설팅에서도 중간 보고는 고객에게 진행 방향을 확인받고 조기에 리스크를 조정하는 기회. 결과물보다 진행 과정의 투명성이 중요
- **발표 Q&A 대비:** 점검 방법론, 관련 법령, 위험관리 3가지는 언제든 1분 내외로 구조화해서 답변할 수 있도록 준비
