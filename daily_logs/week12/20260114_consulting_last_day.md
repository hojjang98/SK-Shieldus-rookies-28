```

→ 전사 업무 분류기준 및 업무 매뉴얼 검토

   추진 동의서 및 신청서 양식 확인

→ 인터뷰 결과 정리
→ 개인정보 처리 양식 사용 현황 확인

4단계: 개인정보 흐름표/흐름도 검토 및 보완
→ 관련 부서 위 담당자 검토 및 보완
→ 수행인력 검토 및 보완

```

```
   └─ 고객정보 시스템 접근
   └─ 대상 고객 조회 및 추출

2. 외부 관리 시스템을 위해 고객정보 일괄전송 후 발송
   └─ 고객정보 조회 시스템으로 전송
   └─ 고객정보 DB에서 정보 추출

3. SMS 발송 업체로 전송
   └─ 교객ID, 고객명, 주소 전송
   └─ SMS 발송

보안 고려사항:
→ 담당자는 고객에게 고객정보를 위해 전자결제 시스템으로 고객정보를 전성하여 처리
→ 고객정보 서비스 팀으로 고객정보 일괄등록 후 발송
→ 고객정보 DB 일괄전송 시 로그 기록 의무
→ 파일 피일 사제
```

### 개인정보 흐름도 체크포인트

**수집 단계:**
- 수집경로 분석
- 필요 정보 수집 여부 및 과도한 개인정보 수집 여부 파악
- 고객정보 관리부서 담당자 대상 설문조사 진행

**저장 단계:**
- 개인정보 서류 보관 방법
- 개인정보 DB 접근 통제 및 백업 여부
- 계정권한 통제 및 신청서 양식 검증

**이용 단계:**
- 타 시스템 연동 여부 및 보호책 수립
- 전송 시 암호화 여부

**제공 단계:**
- 온/오프라인 전송 방법
- 전송 시 암호화 여부

**파기 단계:**

**자산 관리 담당자와의 인터뷰를 통해:**


**부서별 담당자 인터뷰:**

```
분류 체계:
```

## 5. 보호대책 수립

### 보호대책 전략 수립

**전략 수립 절차:**

```
고객 환경 → 점검 결과 → 우선순위 결정 → 위험관리 계획
```

**우선순위 결정 기준:**

| 구분 | 보안 위험도 | 구축 난이도 | 수행 난이도 |

| **수행 난이도** | 소요 시간 | - | - |

### 프로젝트 1년 이내 전략 수립

**즉시 실행 항목:**
- 도출된 문제점 중 처방적인 위험을 담고 조치가 이루어져야 함

**단기 실행 항목:**
- 신규 투자 없이 각 자원의 기본 기능을 활용하여 구현
- 단기간에 구현과 조치가 가능한 독자적인 기능 구현

**장기 실행 항목:**
- 심해사고 발생시 파급효과 및 업무 중요도가 높은 시스템 보안 강화

### 중요도로 정책/지침/절차/매뉴얼 체계 수립

**법 준거성 검토:**

| 구분 | 내용 | 실무 적용 |
|:---|:---|:---|
| **신규 컴플라이언스 요건 반영** | 정보보안 관련 법령 및 감독기관 추가 요구사항 반영 | 개인정보보호법, ISMS-P 개정사항 반영 |
| **내부 정보보안 변화요인 반영** | 국내외 표준의 개선항목 요건 반영 | ISO27001 2022 개정 반영 |
| **내부 정보보안 변화요인 반영** | 내부 조직 및 담당부서 변경 | 조직 개편 시 역할 재정의 |
| | 보안 강화 차원의 추가 준수사항 | 제로 트러스트 등 신규 보안 요구사항 |
| | IT인프라 및 신규 기술 도입 환경 | 클라우드, 컨테이너 환경 정책 추가 |

**정책/지침/절차간 연계 일관성 유지:**

```
정책 체계:
1. 정책 (Policy)
   - 조직의 정보보호 방향과 원칙
   - 경영진 승인 필요
   - 전사적 적용 범위

2. 지침 (Guideline)
   - 정책 실행을 위한 구체적 기준
   - 부서별 적용 방법 명시

3. 절차 (Procedure)
   - 지침 이행을 위한 단계별 수행 방법
   - 업무 프로세스와 연계

4. 매뉴얼 (Manual)
   - 절차 수행을 위한 상세 작업 지침
   - 시스템별 설정 방법 등
```

### 업무환경에 부합된 정책

**업무환경과 불일치 또는 누락된 개인정보보호 정책 재개정:**
- 업무환경에 부합된 정책
- 과 정책과 불일치 또는 누락된 개인정보보호 정책 재개정

**이행 불가한 비현실적 항목 배제:**
- 변화된 현실을 받영하지 못하고, 통제를 위한 통제 향목 등 삭제
- 업무 특성에 맞는 것을 불허고 비현실적 항목에 대해서는 적용 제외 등 방안 협의

**정보보안 조직 구성 및 업무 프로세스 개선안:**
- 개인정보보호 업무 조직, 담당자 명시
- 개인정보보호 업무 프로세스 분석 설계
- 업무양식 표준화

## 9. 배운 점 및 인사이트

### 새로 알게 된 점

- **ISMS-P 통합 인증 체계:** 정보보호와 개인정보보호를 통합한 101개 통제항목이 단순 체크리스트가 아니라 조직의 보안 성숙도를 높이는 프레임워크임을 이해
- **환경분석의 중요성:** 컨설팅의 시작은 문서가 아닌 실제 현장의 조직/업무/시스템 현황 파악이며, 이것이 전체 프로젝트의 방향을 결정
- **흐름도 작성의 실무적 의미:** 정보서비스 흐름도와 개인정보 흐름도는 단순 다이어그램이 아니라 실제 담당자 인터뷰를 통해 현실과 문서의 Gap을 발견하는 도구
- **자산 중요도 평가 방법론:** 기밀성/무결성/가용성 3가지 측면에서 자산을 1~3등급으로 분류하고, 1등급 자산에 집중한 보호대책 수립
- **보안컨설팅 사업의 구조:** 정형(ISMS-P, 취약점 진단)과 비정형(개발보안, 클라우드 보안) 컨설팅의 차이와 각각의 접근 방법

### 이전 학습과의 연결고리

- **웹 취약점 진단 (Day 53)과 연계:** 어제 실습한 SQL Injection, Path Traversal 같은 기술적 취약점이 오늘 배운 위험평가의 '취약점 진단' 단계에서 실제로 점검되는 항목
- **ISMS 컨설팅 개념 확장:** Day 51에서 배운 ISMS 컨설팅 프로세스가 오늘 학습한 환경분석 → 위험평가 → 보호대책 수립 → 인증심사 전체 사이클로 구체화
- **DNA Lab 프로젝트 연결:** 자동화 취약점 스캐너가 오늘 배운 '인프라 취약점 진단' 단계의 일부를 자동화하는 도구임을 인지
- **개인정보보호법 이해:** 웹 개발 시 수집하는 개인정보가 오늘 배운 '수집→저장→이용→제공→파기' 생명주기 관리 대상임을 연결

### 실무 적용 아이디어

**SOC 분석가 관점:**
- **정보서비스 흐름도 활용:** SOC에서 탐지한 이상 트래픽을 분석할 때 정보서비스 흐름도를 참조하면 정상 업무 흐름과 비정상 흐름을 구분 가능

- **개인정보 흐름도 기반 DLP 정책:** 개인정보 흐름도에 명시되지 않은 경로로의 개인정보 전송 시도를 DLP로 차단

**보안 컨설턴트 준비:**
- **환경분석 템플릿 구축:** 오늘 배운 조직/업무/IT 환경분석 체크리스트를 기반으로 인터뷰 질문지 템플릿 작성
- **흐름도 작성 도구 숙달:** Visio, Draw.io 등 다이어그램 도구로 정보서비스 흐름도와 개인정보 흐름도 작성 연습
- **보고서 작성 능력:** 취약점 진단 결과를 경영진이 이해할 수 있는 요약본과 기술팀이 조치할 수 있는 상세본으로 구분하여 작성

**개인 학습 방향:**
- **ISMS-P 인증기준 101개 항목 숙지:** 프로젝트 파일의 ISMS-P 안내서를 정독하여 각 통제항목의 의미와 점검 방법 학습
- **금융권 ISMS-P 안내서 활용:** 금융보안원 안내서의 세부 통제항목을 통해 금융권 특화 요구사항 파악
- **실전 흐름도 작성 연습:** 가상의 쇼핑몰 시나리오를 만들어 정보서비스 흐름도와 개인정보 흐름도 작성 실습

**Today's Insight:**

오늘은 ISMS-P 인증제도의 실무적 측면을 깊이 있게 학습하면서 보안컨설팅이 단순히 취약점을 찾는 것이 아니라 조직의 보안 관리체계 전체를 설계하고 구축하는 프로세스임을 이해했다. 특히 환경분석 단계에서 조직/업무/IT 환경을 파악하고, 담당자 인터뷰를 통해 실제 현황과 문서의 Gap을 발견하는 과정이 컨설팅의 핵심임을 깨달았다. 정보서비스 흐름도와 개인정보 흐름도는 단순한 다이어그램이 아니라 조직의 정보 흐름을 가시화하여 보안 취약점을 식별하는 강력한 도구다. 자산 중요도 평가를 통해 모든 자산에 동일한 보안 수준을 적용하는 것이 아니라, 1등급 자산에 집중하여 효율적인 보안 투자를 하는 것이 실무적 접근임을 배웠다. Day 53에서 학습한 SQL Injection, Path Traversal 같은 기술적 취약점이 오늘 배운 위험평가의 '취약점 진단' 단계에서 실제로 점검되며, 이것이 보호대책 수립으로 이어지는 전체 사이클을 이해하게 되었다. 보안컨설턴트는 기술적 지식뿐만 아니라 조직 분석, 인터뷰 스킬, 문서 작성 능력이 필수이며, 이를 위해 ISMS-P 인증기준 101개 항목과 금융권 세부 통제항목을 체계적으로 학습해야겠다는 목표가 명확해졌다.
