## 📘 개요 (Overview)
**Mini Security Log Monitor**는 1주차 학습 내용을 종합하여 구현한 “보안 이벤트 감지 및 기록 시스템”의 기본 형태이다.  

이 프로젝트는 파이썬의 기초 문법(제어문, 함수, 클래스)을 보안적 사고로 재구성하여 **입력 검증 → 탐지 → 예외 처리 → 감사 로깅**의 흐름을 구현하는 것을 목표로 한다.  

---

## 🎯 목적 (Objective)
- 문자열 처리와 조건문을 이용해 **로그 이벤트 탐지 로직**을 구성한다.  
- 데코레이터를 통해 **공통 로깅·권한 검증 로직**을 자동화한다.  
- 예외 발생 시 중단 없이 처리되는 **Resilient 코드 구조**를 구현한다.  
- JSON 기반의 데이터 구조를 통해 **보안 감사 로그 형식화**를 실습한다.  

---

## ⚙️ 주요 기능 (Key Features)
| 기능 | 설명 |
|------|------|
| ✅ 사용자 인증 검증 | 인증되지 않은 사용자의 접근을 차단 |
| 🧾 이벤트 탐지 및 기록 | 로그 파일 내 `ERROR`, `UNAUTHORIZED` 등 이벤트 감지 및 저장 |
| 🔒 사용자 마스킹 처리 | 개인정보 보호를 위한 이름 비식별화 (`ad***`) |
| ⚠️ 예외 처리 및 경고 | 오류 발생 시 `try-except`로 중단 없이 로깅 |
| 🧩 데코레이터 기반 구조 | 공통 보안 로직(로깅, 권한 체크)을 함수 외부에서 관리 |

## 🧠 학습 연결 (Learning Link)
| 학습 내용 | 적용 포인트 |
|------------|-------------|
| `dict.get()` | KeyError 방지 및 안전한 데이터 접근 |

| `@decorator` | 접근 감사 로직의 공통화 |
| `try-except` | 예외 발생 시 안정적 흐름 유지 |
| `with open()` | 로그 파일 입출력의 자원 안정성 확보 |
| 문자열 슬라이싱 | 사용자 마스킹(`ab***`) 처리 |

## 🔐 실무 연계 (SOC Perspective)
이 프로젝트는 **보안관제(SOC)**의 4단계 구조를 간소화하여 코드 레벨로 실현한 예시다.

| SOC 단계 | 코드 내 대응 기능 |
|-----------|------------------|
| 로그 수집 (Collection) | `system.log` 파일을 읽어 이벤트 수집 |
| 이벤트 탐지 (Detection) | 문자열 탐색과 조건문으로 이상 행위 감지 |
| 이벤트 기록 (Logging) | JSON 기반 `access.log`에 결과 저장 |
| 감사 및 분석 (Audit) | 마스킹된 사용자 로그로 접근 내역 확인 |

> 💡 학습 수준에 맞춰, 실제 관제 시스템의 “수집 → 탐지 → 기록 → 감사” 단계를 축약해 구성한 형태다.

## 실행 방법 (How to Run)

1. system.log 준비

```bash
2025-10-31 09:00 [INFO] System boot completed
2025-10-31 09:12 [ERROR] Connection timeout
2025-10-31 10:25 [ERROR] Unauthorized access attempt
```

2. 실행

```bash
python secure_log_monitor.py
```

3. 결과 확인

```bash
[INFO] 접근 성공: ad**
[WARNING] 접근 실패: ca** (PermissionError)
```

- access.log 파일이 자동 생성되어 JSON 형식으로 결과가 기록된다.
