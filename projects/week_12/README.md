# Week 12: 보안 컨설팅 프로젝트 (4주 계획)

## 프로젝트 배경

파이널 프로젝트 주제를 정하는 과정에서 관제(SOC)와 보안 컨설팅 사이에서 고민이 있었다. 
미니 프로젝트 2에서 취약점 진단을 경험했지만, 당시에는 팀 프로젝트로 진행되어 각자 맡은 부분이 달랐다. 
컨설팅은 따로 주간 과제로 진행하긴 했으나, 진단부터 개선, 모니터링까지 전체 프로세스를 혼자서 A to Z로 경험해보고 싶었다.

이번 프로젝트에서는 보안 컨설팅의 전체 프로세스를 단계별로 경험하며, 
취약한 시스템 구축 → 진단 → 개선 → 모니터링까지 실무 컨설턴트의 업무 흐름을 직접 수행해본다.

## 프로젝트 개요
중소기업 E-Commerce 보안 컨설팅 시뮬레이션 프로젝트

## 전체 일정
- **Week 12 (현재)**: 취약한 시스템 구축
- **Week 13**: 취약점 진단 (모의해킹 + 취약점 스캔)
- **Week 14**: 보안 개선 구현
- **Week 15**: 모니터링 체계 + 최종 컨설팅 보고서

## Week 1: 취약한 쇼핑몰 구축

### 구현 기능
- 회원가입/로그인
- 상품 목록 조회
- 장바구니
- 주문/결제
- 프로필 관리

### 의도적으로 삽입한 취약점
1. **SQL Injection**: 로그인, 상품 검색
2. **XSS (Cross-Site Scripting)**: 댓글, 프로필
3. **인증 우회**: 세션 관리 취약
4. **파일 업로드 취약점**: 프로필 이미지 업로드

### 실행 방법
```bash
cd ecommerce_vulnerable
python init_db.py          # DB 초기화
python app.py              # 서버 실행
```

### 접속
- URL: http://localhost:5000
- 테스트 계정: admin / admin123
