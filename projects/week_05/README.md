# 리눅스 파일 시스템과 권한 관리

## 프로젝트 개요

본 프로젝트는 리눅스 파일 시스템 아키텍처와 권한 관리 메커니즘에 대한 종합적인 기술 분석을 제공합니다. 실습 중심의 개발 프로젝트보다는, 리눅스의 핵심 개념들을 체계적으로 정리한 **기술 보고서** 형태로 작성하여 시스템 관리 및 보안 실무를 위한 참고 문서로 활용할 수 있도록 구성했습니다.

---

## 배경

Week 5 커리큘럼에서는 리눅스와 네트워크의 기초 개념을 중점적으로 학습했습니다. 이러한 주제들은 시스템 수준의 이해에 필수적이지만, 학습 범위가 기본 원리에 집중되어 있어 일반적인 코딩 프로젝트로 구현하기에는 적합하지 않았습니다.

학습 효과를 극대화하기 위해, 본 프로젝트는 **기술 문서화**에 초점을 맞춰 리눅스 파일 시스템과 권한 관리의 핵심 개념, 모범 사례, 보안 고려사항을 체계적으로 정리했습니다.

---

## 프로젝트 목표

- 리눅스 파일 시스템 계층 구조와 디렉토리 구성에 대한 깊이 있는 이해 확립
- 리눅스 환경에서의 다양한 파일 타입과 특성 문서화
- 멀티유저 환경에서 권한 관리 시스템의 역할과 보안적 의미 분석
- 권한 관리 모범 사례 및 실제 보안 시나리오 파악
- 시스템 관리자 및 개발자를 위한 실무 가이드라인 제공

---

## 주요 다루는 내용

### 1. 리눅스 파일 시스템 구조
- 루트 디렉토리와 계층적 트리 구조
- 마운트 개념과 파일시스템 통합
- 표준 디렉토리의 역할 (/bin, /etc, /home, /var, /dev 등)
- 절대 경로와 상대 경로

### 2. 파일 타입과 특성
- 일반 파일, 디렉토리, 심볼릭 링크
- 특수 파일: 블록 장치, 문자 장치, 파이프, 소켓
- inode 구조와 파일 메타데이터 관리
- 하드 링크 vs 심볼릭 링크

### 3. 권한 관리 시스템
- 읽기(r), 쓰기(w), 실행(x) 권한
- 소유자, 그룹, 기타 사용자 권한 모델
- 권한 표현: 심볼릭 모드와 숫자 모드
- chmod, chown, chgrp 명령어

### 4. 보안 관점
- 최소 권한 원칙 (Principle of Least Privilege)
- 특수 권한: SetUID, SetGID, Sticky Bit
- 보안 취약점과 권한 상승 위험
- 권한 감사 및 시스템 강화

### 5. 실무 활용
- 파일 및 디렉토리 권한 모범 사례
- 파일 타입별 일반적인 권한 설정
- 시스템 감사 기법 및 보안 점검

## 핵심 인사이트

### 왜 문서화 중심인가?

본 프로젝트가 의도적으로 문서화 중심 접근을 택한 이유:

1. **기초 다지기**: 리눅스 파일 시스템과 권한은 피상적인 구현보다 체계적인 문서화를 통한 학습이 효과적인 기초 개념입니다
2. **참고 가치**: 잘 구조화된 기술 보고서는 향후 프로젝트와 문제 해결 시 장기적인 참고 자료로 활용됩니다
3. **전문성 향상**: 기술 문서 작성은 시스템 관리자와 DevOps 엔지니어에게 필수적인 역량입니다
4. **지식 정착**: 개념을 일관된 문서로 정리하는 과정이 이해도와 기억 정착에 도움이 됩니다

### 실무 적용 분야

본 보고서에서 다룬 개념들은 다음 분야에 직접 적용됩니다:
- 서버 관리 및 설정
- Docker 컨테이너 보안 (사용자 권한, 볼륨 마운트)
- CI/CD 파이프라인 보안 (빌드 사용자 권한)
- 클라우드 인프라 관리 (IAM 및 파일 수준 권한)
- 보안 감사 및 컴플라이언스

## 향후 확장 방향

본 작업을 확장할 수 있는 분야:

- Access Control Lists (ACL)을 통한 세밀한 권한 제어
- SELinux 및 AppArmor 보안 프레임워크
- 파일시스템 타입별 특성 (ext4, XFS, Btrfs)
- 고급 주제: capabilities, namespaces, cgroups
- 권한 감사 및 수정 자동화 스크립트
