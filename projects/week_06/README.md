# TCP/IP 네트워크 프로토콜 분석 및 실무 적용

## 프로젝트 개요

본 프로젝트는 TCP/IP 네트워크 교육을 통해 학습한 내용을 체계적으로 정리하고, 실무 관점에서 프로토콜의 동작 원리와 보안 개념을 심화하기 위해 작성된 **기술 보고서**입니다. 이 보고서는 시스템 관리 및 네트워크 보안 실무를 위한 참고 문서로 활용될 수 있도록 구성되었습니다.

---

## 배경

현대 IT 인프라의 기반이 되는 **네트워크 기술**은 시스템 관리자와 보안 전문가에게 필수적인 지식입니다. 서버가 아무리 안정적으로 구축되어 있어도 네트워크 통신이 원활하지 않으면 서비스를 제공할 수 없으며, 네트워크 계층의 문제는 전체 시스템의 가용성에 직접적인 영향을 미칩니다.

학습 효과를 극대화하기 위해, 본 프로젝트는 TCP/IP 네트워크 교육을 통해 학습한 내용을 **기술 문서화**에 초점을 맞춰 체계적으로 정리했습니다.

## 주요 다루는 내용

### 1. 네트워크 계층 모델

* **계층화의 필요성:** 네트워크 통신은 복잡하여, 이를 효율적으로 관리하고 표준화하기 위해 계층화 모델이 도입되었습니다. 계층화는 각 계층의 독립적 동작을 보장하며, 유지보수를 용이하게 만듭니다.
* **모델 비교:** OSI 7 계층은 이론적인 참조 모델이며, TCP/IP 4 계층은 실제 인터넷에서 사용되는 프로토콜 스택입니다. 실무에서는 TCP/IP 모델이 더 실용적입니다.

### 2. 주요 프로토콜의 동작 원리

* **ARP (Address Resolution Protocol):** 같은 네트워크 내에서 IP 주소를 통해 목적지의 MAC 주소를 찾아주는 프로토콜입니다.
* **ICMP (Internet Control Message Protocol):** IP 프로토콜의 보조 역할을 하며, 네트워크 상태 진단 및 오류를 보고하는 제어 및 관리용 프로토콜입니다.
* **TCP 3-Way Handshake:** TCP 연결을 수립하는 과정으로, SYN -> SYN-ACK -> ACK의 3단계를 거치며 신뢰성 있는 데이터 전송을 준비합니다.
* **HTTPS와 TLS:** HTTPS는 HTTP에 보안 계층 (TLS/SSL)을 추가한 것으로, 데이터를 암호화, 무결성을 보장하며 서버의 신원을 인증합니다.

### 3. 네트워크 보안 기초

* **방화벽의 역할:** 미리 정의된 보안 규칙에 따라 네트워크 트래픽을 허용하거나 차단하며, 내부 네트워크를 외부 위협으로부터 보호합니다.
* **IDS (Intrusion Detection System) vs IPS (Intrusion Prevention System):** IDS는 침입 탐지 및 경고를 담당하는 **수동적** 시스템이며, IPS는 침입을 실시간으로 탐지하고 차단하는 **능동적** 시스템입니다.
* **포트 스캔 보안:** 포트 스캔은 열린 포트를 탐지하는 기술이므로, 불필요한 서비스는 중지하고 필요한 포트만 선택적으로 열어두는 것이 중요합니다.

### 4. 실무 적용 및 진단

* **네트워크 문제 진단 절차:** 실무에서 문제가 발생했을 때 물리/데이터링크 계층부터 응용 계층까지 **계층별로 접근**하여 원인을 파악합니다.
* **실무 도구 활용:** Wireshark, nmap, Packet Tracer, 그리고 Linux 명령어 (ping, traceroute, netstat, ss, ip, tcpdump) 등이 활용됩니다.
